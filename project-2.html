AnimaLook - Web

Category : UX - UI / Web
Completion : Dec 2018
Sector : Pet adoption

With this date came my first ideas to face the final project of Ironhack School.

RESEARCH

I launched surveys and interviewed several users to obtain information about adoptions What problems did they find when adopting? What they needed?â€¦ I faced the following problems:

  • - Users think that there is limited information about adoption procedures on the websites of animal shelters.
  • - Users waste a lot of time and find it cumbersome navigating different shelter websites.
  • - Online adoptions is a really cold process, users need to have close relationships with animal shelters.
  • - 87% want to be advised to find an animal related to their lifestyle.

On the other hand, animal shelters confirmed the problem of knowing the user better before validating the adoption to ensure that they choose related lifestyle dogs and that no further returns occur.

GOAL

To increase the number of pet adoptions and solving the user problems detected in the research. How? through a platform where user can find animals of different pet shelters at a glance.

FOCUS ON USER

First I made a Competitive Analysis and confirmed the nonexistence of similar products. In addition, the shelter websites lack clear information about procedures and the hierarchy of information is not optimally available. Tools like Lean UX Canvas helped me to organize the concepts and start shaping all the information.

User Persona

Tools like Emphaty Map helped me get closer to the user and know what he thinks, hears, says and sees. This way I was getting closer and closer to building the right product for the user.

User Journey

It is about tracing the path that user follows when interacting with the product, reflecting his emotional state in each step.

Previous:

Through this tool came user flow in addition to identifying pain / gain points and setting some goals to face them. Previous User journey shows the actual user flow identifying pain points; however, the Future User journey tries to re-imagine what the new user - product interaction would be like by reflecting the improved pain points, some now converted into gain points and others that will help identify design opportunities.

Future:

Affinity diagram helped me identify design opportunities. Through Moscow and Complexity Value I studied which features were business view viable and discarded those that were not. Card sorting and testing low fidelity finally defined the flow.

FROM UX TO UI

It is a product that reflects transparency, how? offering information of any process and giving to user the control of their actions; professional, without ceasing to be close since we talk about such an important decision that could change the user's life but at the same time warm, close and trusting since we deal with animals.

This is how moodboard emerges and determines part of the style like colors, shapes, illustration or texture.

The typography chosen is Imperator. Serifa to achieve readability and offer strength since the product contains cards with a lot of information.
